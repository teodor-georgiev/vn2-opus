=== PAGE 39 ===
Reinforcement Learning for Inventory Decisions
REINFORCE algorithm
etatS
noitcA
Vanilla policy gradient (policy-based RL):
learn a stochastic policy that maps state → action
Training loop:
Run rollouts with the current policy
Collect (state, action, reward)
Compute episode Returns
Update neural network weights to maximize expected return
Outputs (parameters of the action distribution):
discrete: action probabilities (softmax)
continuous: parameters of an action distribution (like mean + std)
Why a distribution?
exploration is built-in: the agent must sample actions to try different decisions
Main drawback: Returns come from full episodes, so many random factors add noise
→ high-variance gradients, slower and less stable training

=== PAGE 41 ===
Reinforcement Learning for Inventory Decisions
PPO (Proximal Policy Optimization)
PPO stabilizes policy updates by limiting how much the policy can change per update
Clipped objectives:
prevents “too large” updates when the new policy deviates too much from the old one
more stable learning
GAE (Generalized Advantage Estimation):
computes a low-noise advantage signal by mixing short- and long-term returns
Why it is better than REINFORCE:
lower-variance learning signal + safer updates

What the actions mean
Inventory projection
Inputs:
I0 - On hand
T1, T2 - In transit
d1, d2, d3 - Forecasted Demand
Project inventory:
E1 = max( I0 + T1 - d1, 0 )
E2 = max( E1 + T2 - d2, 0 )
Order Estimation Policy outputs two actions:
multiplier ( between [0, 2] → tanh + 1.0 )
buffer (non-negative → softplus )
base = max( d3 - E2 , 0 )
order = round( max( base × multiplier + buffer × scale_factor , 0) )

=== PAGE 50 ===
Reinforcement Learning for Inventory Decisions
RL Training Setup
1) Pretrained Demand Neural Network
Frozen demand components provide 3-week demand forecasts
2) Training Episodes Simulator - Each epoch samples:
random initial inventory state
random episode start date (only from training set)
3) Build observations from Simulator & Forecast:
Observation = [ on-hand inventory ( I0 ) + in-transit ( T1, T2 ) + predicted demand ( d1, d2, d3 ) ]
All scaled by scale factor
4) Roll out a stochastic policy
Policy outputs mean + std
Action is sampled from a squashed Gaussian Policy
5) Reward
Reward = negative inventory cost = - ( holding cost + shortage cost )
6) Training with PPO + GAE
Collect trajectories → advantages & returns
Policy updated with clipped PPO objective + value loss + entropy bonus
7) Validation
Use deterministic action from the policy mean
Early Stopping monitors validation total cost and selects the best checkpoint
